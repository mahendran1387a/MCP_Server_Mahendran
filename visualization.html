üéØ Key Architecture Decisions

1. Single ChromaDB Connection
Why: Prevents database locking
How: Only web_server.py opens ChromaDB
Impact: No more "Connection closed" errors
2. HTTP API for RAG
Why: MCP server needs RAG access
How: POST to /api/rag/query endpoint
Impact: Shared RAG without database conflicts
3. AsyncClientManager
Why: Bridge sync Flask ‚Üî async MCP
How: Background thread with event loop
Impact: Smooth async/sync integration

üìä Complete Request Flow

  1. User asks question in browser
  2. Flask receives HTTP POST to /api/query
  3. AsyncClientManager routes to MCP client
  4. MCP Client sends to Ollama with tool descriptions
  5. Ollama analyzes and decides which tool to use
  6. MCP Client calls MCP server with tool request
  7. MCP Server executes appropriate tool
  8. RAG Tool (if used) calls HTTP API ‚Üí web server ‚Üí ChromaDB
  9. Other Tools execute directly (calculator, weather, etc.)
  10. Tool Result returns to MCP client
  11. Ollama formats result into natural language
  12. Response bubbles back through all layers to browser
